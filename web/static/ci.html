Workflow Strategy Insight

The RobotJudge-CI pipeline implements a multi-stage validation strategy using the pytest framework to ensure absolute solver integrity.

1

Environment & Test Generation

The pipeline initializes a clean, isolated Docker environment. It programmatically generates the required test suite to ensure no data stale-state exists during evaluation.

2

Formal Schema Validation

Every testcase and configuration file is processed through a strict JSON Schema validator. This ensures full structural compliance before execution begins.

3

Pytest Orchestrated Validation

The core judge logic is subjected to pytest suites. We verify that reference heuristics (A*, Dijkstra) perform within established cost and time tolerances.

4

Baseline Regression & Determinism

Final-stage verification ensures that new changes do not introduce non-deterministic search behaviors or regressive metrics compared to the "Golden Master" data.
